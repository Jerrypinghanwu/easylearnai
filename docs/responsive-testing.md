# 響應式設計測試結果

## 測試設備
- iPhone 14（iOS 17）：390×844、橫向 844×390
- iPad Air（iPadOS 17）：820×1180、橫向 1180×820
- Pixel 7（Android 14）：412×915、橫向 915×412
- Surface Pro（Windows 11）：1024×1366、橫向 1366×1024

## 驗證項目
- 漢堡式選單：手機與平板顯示切換正常，點擊可開合；選單連結點擊自動收合。
- 文字排版：標題、副標、段落在不同解析度下字級與行距可讀性佳。
- 圖片媒體：`img { max-width:100% }` 生效，圖片自動縮放；重要圖片使用 `loading="lazy"` 與 `decoding="async"`。
- 表單可用性：輸入框高度 ≥44px、足夠間距；Email 欄位使用 `type=email` 與自動填寫。
- 方向切換：橫向與縱向皆能正常重排並顯示。

## Chrome DevTools 測試
- 使用裝置模擬器切換常見尺寸，視覺與互動一致。
- 無版面溢出（horizontal scroll）問題。
